## Active Logging Rules

- For new tasks, record lifecycle in this strict order: `assigned` -> `started` -> `file-touched` -> `validation` -> `completed` or `blocker`.
- New task delegation: log immediately as `assigned` and include assignee Vault ID/tool ID.
- Completion: log when the agent responds with results using `completed`.
- File touch: log each touched/changed file path explicitly in `Files touched` using `file-touched`.
- Shared-file updates: for new logs, use `file-touched` (legacy `file-changed` entries remain in history).
- Validation: log as `validation` with both preview URL and pass/fail status in the result when applicable.
- This log should stay append-only; avoid rewriting historical rows.
